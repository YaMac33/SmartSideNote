🔔 1. LINE / LINE WORKS に GitHub通知を送る

✅ 実現方法（どちらでも共通）：

  • GitHubのWebhook機能を使って、特定イベント（push、PRなど）を検知
  • LINEまたはLINE WORKSのBotへHTTP POSTリクエストで通知

📦 必要なもの：

  • LINE公式アカウント or LINE WORKSのBot（Webhook URL）
  • GitHubリポジトリの設定 → Webhook追加
  • メッセージ整形（JSONで送る）

例（GitHub → LINE）：

[GitHub通知]
リポジトリ: my-project
ユーザー: username
イベント: Pull Request #12 が作成されました！
リンク: https://github.com/your-org/my-project/pull/12

⚙️ 2. GitHub Actions でLINEに通知を送る

✅ 使用例：

  • CIが失敗したらLINE通知
  • デプロイ完了後にLINE通知
  • PRがマージされたら通知

💡 例：GitHub ActionsのYAMLに以下を追加

- name: Notify LINE
  run: |
    curl -X POST https://api.line.me/v2/bot/message/push \
      -H "Authorization: Bearer ${{ secrets.LINE_TOKEN }}" \
      -H "Content-Type: application/json" \
      -d '{
        "to": "ユーザーIDまたはグループID",
        "messages": [{"type": "text", "text": "CI/CDが成功しました！"}]
      }'

🔑 LINEトークンなどはGitHub Secretsに登録しておくと安全です。

🧠 3. LINE Bot から GitHubの情報を取得・操作

LINE Botにメッセージを送ると、GitHubのAPIを叩いて情報を返すようにできます。

✅ 例：

  • ユーザー「PR一覧見せて」
  • Bot「以下がオープン中のPRです：...」

📦 必要な知識：

  • LINE Messaging API
  • GitHub REST API or GraphQL API
  • サーバー or FaaS（Google Apps Script、Firebase Functionsなど）

💬 4. LINE WORKS 専用の連携方法

✅ 特徴：

  • LINE WORKS BotはWebhook URLが簡単に使える
  • 管理者がBotを組織に追加するだけで、全員に通知可能
  • トークルームへの自動投稿ができる
