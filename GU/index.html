noteとGitHub Pages、自由度が高いのはどっち？

HTML埋め込みの観点から徹底比較

ブログ投稿サイトで、「noteはHTMLを埋め込むことができないから自由度が低い」と感じることはありませんか？一方で、「GitHub PagesのほうがHTMLを埋め込めるから自由度が高い」という意見もよく聞きます。

あなたのその認識は概ね正しいです。ブログ投稿プラットフォームであるnoteと、静的サイトホスティングサービスであるGitHub Pagesは、それぞれ異なる目的と特性を持っているため、「自由度」という観点ではGitHub Pagesに軍配が上がります。

noteの特徴と「自由度」の制約

noteは、誰でも簡単に文章や画像を投稿できることに特化したサービスです。そのため、プラットフォームとしての利便性を高めるために、以下のような制約が設けられています。

  • HTMLの直接編集は不可: noteの編集画面は、あらかじめ用意された機能（見出し、画像、動画の埋め込みなど）を使ってコンテンツを作成する仕組みです。ユーザーが直接HTMLタグを記述することはできません。これにより、セキュリティリスクの低減や、サイト全体の一貫したデザインを維持しています。
  • 埋め込み（iframe）は可能: 特定の外部サービス（YouTube、Twitter、Spotifyなど）のコンテンツは、URLを貼り付けることで埋め込むことができます。しかし、これもnoteが許可したサービスに限られます。
  • 目的: noteは「コンテンツを簡単に発信し、多くの人に届ける」ことに主眼を置いています。HTMLの知識がなくてもブログを始められる点が最大のメリットです。

GitHub Pagesの特徴と「自由度」の高さ

一方、GitHub Pagesは、GitHubのリポジトリに保存されたHTML、CSS、JavaScriptなどのファイルをそのままウェブサイトとして公開するサービスです。

  • HTMLを自由に記述可能: あなたが作成したHTMLファイルをそのまま公開できるため、レイアウト、デザイン、機能など、ウェブサイトのすべてを自由にカスタマイズできます。これが「自由度が高い」と感じる最大の理由です。
  • 静的サイトのホスティング: GitHub Pagesは静的なウェブサイト（データベースやサーバーサイドの処理を必要としないサイト）の公開に特化しています。ブログ記事の更新も、HTMLファイルを更新してリポジトリにプッシュすれば完了です。
  • 自動化が可能: GitHub Actionsを使えば、RSSフィードの取得や記事の自動生成など、さまざまなタスクを自動化できます。

どちらを選ぶべきか？

どちらが優れているかというよりは、あなたの目的によって使い分けるのが良いでしょう。

noteがおすすめな人

  • 手軽に文章を発信したい。
  • HTMLやCSSの知識がない。
  • noteのコミュニティ内で記事を読まれたい。
  • 有料コンテンツやメンバーシップ機能を利用したい。
  • とにかく早くブログを始めたい。

GitHub Pagesがおすすめな人

  • ウェブサイトの見た目や機能を徹底的にカスタマイズしたい。
  • HTMLやCSS、JavaScriptの知識がある、またはこれから学びたい。
  • ブログ記事の生成やデプロイを自動化したい。
  • ブログを自分の技術ポートフォリオの一部として活用したい。

結論として、HTMLを自由に埋め込みたい、またはウェブサイトの見た目や機能を完全にコントロールしたいのであれば、GitHub Pagesのほうが圧倒的に高い自由度を提供します。noteは手軽さが強みであり、その手軽さと引き換えに自由度が制限されている、という位置づけです。
