動的なサイトは、ユーザーが見る「フロントエンド」と、裏でデータを処理する「バックエンド」の2つが連携して動きます。

  • フロントエンド (Frontend): ユーザーが操作する画面 (HTML, CSS, JS)。バックエンドに処理を依頼し、結果を表示します。
  • バックエンド (Backend): サーバーで動く裏側の処理 (Node.js, Pythonなど)。ユーザー登録やログイン認証など、データの保存や管理を担当します。

バックエンド開発ステップ 🚀

バックエンドの言語を選択してください。フロントエンドや利用サービスは共通です。

1開発環境をReplitに集約する

テンプレートはNode.jsを選択し、index.html, style.css, script.jsとバックエンド用のindex.jsを管理します。

2バックエンドの骨組みを作る (API)

ExpressフレームワークでWebサーバーを立て、/api/registerなどのURL（エンドポイント）を設計します。

3データベースを用意する (Render)

Renderの無料PostgreSQLデータベースを準備し、「接続情報URL」を控えます。

4バックエンドとデータベースをつなぐ

データベース接続URLはReplitのSecrets機能に保存。/api/registerが呼ばれたらDBに保存する処理を実装します。

5フロントエンドとバックエンドを連携させる

JavaScriptのfetch()関数を使い、フロントエンドのフォームからバックエンドのAPIを呼び出します。

6Renderへデプロイし、公開する

ReplitのコードをGitHubに保存後、RenderでWebサービスを作成し連携。Start Commandはnode index.jsです。

1開発環境をReplitに集約する

テンプレートはPython（またはFlask）を選択。ファイル構成はFlaskの作法に合わせ、main.py、templates/index.html、static/style.cssのように配置します。

2バックエンドの骨組みを作る (API)

pip install FlaskでFlaskをインストール。main.pyにサーバーコードを書きます。

from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route('/api/register', methods=['POST'])
def register_user():
    data = request.get_json()
    username = data.get('username')
    # (データベース保存処理)
    print(f"登録リクエスト受信: {username}")
    return jsonify({"status": "success"}), 201

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)

3データベースを用意する (Render)

このステップはNode.jsの場合と全く同じです。RenderでPostgreSQLを作成し、接続情報を控えます。

4バックエンドとデータベースをつなぐ

pip install SQLAlchemy psycopg2-binaryを実行。DB接続URLをSecretsに保存し、SQLAlchemyを使ってDB操作コードを実装します。

5フロントエンドとバックエンドを連携させる

このステップもNode.jsとほぼ同じ。フロントエンドのscript.jsからfetch()を使い、FlaskのAPIを呼び出します。

6Renderへデプロイし、公開する

Renderの設定がNode.jsと少し異なります。

  • Build Command: pip install -r requirements.txt
  • Start Command: 本番用のGunicornサーバーを使います。gunicorn main:appと設定します。

まとめ

見ての通り、動的なサイトを作るという本質的な流れは、どの技術を使っても変わりません。フロントエンドとバックエンドの連携という中心的な概念を理解することが重要です。

  • JavaScriptが好きなら → フロントとバックを同じ言語で書けるNode.jsがおすすめです。
  • Pythonが好きなら → 将来的に機械学習など他の分野への応用も視野に入れやすいPythonがおすすめです。
