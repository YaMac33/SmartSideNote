備忘録：GitHub PagesとCloudflare Pages、静的サイトと動的サイトの違いまとめ 🚀

こんにちは！今回は自分用の備忘録として、GitHub PagesやCloudflare Pagesを使った静的サイトホスティングの違いや、Markdownでのサイト作成、そして個人運営サイトにおける静的・動的サイトの使い分けについて、わかりやすくまとめてみました。

GitHub PagesとCloudflare Pagesって何が違うの？ 🤔

まずは、よく話題に上がるGitHub PagesとCloudflare Pagesの違いから。どちらも無料で使える静的サイトホスティングサービスですが、特徴や使い勝手が結構違います。

GitHub Pagesの特徴

  • GitHubのリポジトリにある静的ファイルをそのまま公開できるサービス。
  • Jekyllという静的サイトジェネレーター（SSG）に対応していて、Markdownで書いた記事を自動でHTMLに変換してくれる。
  • 初心者でも比較的簡単に使いやすく、日本語の情報も多いのが嬉しいポイント。
  • GitHub Pages公式

Cloudflare Pagesの特徴

  • Cloudflareの高速グローバルCDNを使った静的サイトホスティングサービス。
  • GitHubのほかGitLabやBitbucketなど複数のGitリポジトリと連携可能。
  • Node.js環境で自由にビルドツールを使えるので、Next.jsやGatsby、Nuxt.jsなどモダンなフレームワークもラクラク対応。
  • プルリクエストごとにプレビュー環境が自動生成され、チーム開発に便利。
  • Cloudflare Workersというサーバーレス機能も連携できて、動的な処理も拡張できる。
  • ただし、管理画面やドキュメントは英語がメインで、日本語情報はまだ少なめ。
  • Cloudflare Pages公式

ざっくり比較表 📊

項目 GitHub Pages Cloudflare Pages
連携リポジトリ GitHubのみ GitHub, GitLab, Bitbucket対応
ビルド環境 Jekyll標準、他は自分で設定 Node.jsで自由に設定可能
CDN GitHub独自CDN Cloudflareの超高速CDN
プレビュー機能 基本なし プルリクエスト毎に自動生成
動的処理 不可 Cloudflare Workersで可能
日本語情報 豊富 少なめ
初心者向け ◎ △

Markdownで作る静的サイトの自由度って？ 📝

「Markdownで書くなら自由度が低いのでは？」と思うかもしれませんが、実はそうでもありません。Markdownは文章を書くのに便利でシンプルな記法ですが、Markdown内にHTMLタグを直接書くことも可能です。さらに、<style>タグや<script>タグも埋め込めます（環境によって制限はありますが）。

以下はMarkdown内にHTMLとJavaScriptを埋め込んだ簡単なサンプルコードです。

# はじめてのブログ記事

こんにちは！これはMarkdownで書いた記事です。

<button onclick="alert('こんにちは！')">クリックしてね</button>

<style>
button {
  background-color: #4CAF50;
  color: white;
  padding: 8px 16px;
  border: none;
  cursor: pointer;
}
</style>

このコードのポイント：

  • 普通にMarkdownでタイトルや文章を書きつつ、
  • <button>タグを使ってボタンを設置し、JavaScriptのonclickイベントでメッセージを表示。
  • <style>タグでボタンの見た目をカスタマイズ。
  • GitHub PagesのJekyllやCloudflare PagesのようなSSGでは、MarkdownをHTMLに変換するときにこうしたHTMLやスクリプトも一緒に組み込めます。

静的サイトと動的サイト、どっちが必要？ ⚖️

静的サイトが向いているケース

  • ブログや技術メモ、写真ギャラリーなどのコンテンツを主に公開する。
  • 更新は手動で行い、アクセス速度やセキュリティを重視したい。
  • GitHub PagesやCloudflare Pagesのような静的ホスティングサービスで安価に運用したい。

動的サイトが必要になるケース

  • ユーザーごとのログインやマイページがある。
  • コメント投稿や掲示板などユーザーの書き込みをリアルタイムで反映したい。
  • 複雑な検索や予約管理など動的なデータ処理が必要。

動的機能を静的サイトに追加するなら？ 🛠️

静的サイトに動的要素を組み込みたい場合は、外部サービスを組み合わせる方法がおすすめです。

例1：コメント機能（Disqus）

Disqusは簡単にコメント欄を設置できるサービスです。以下のコードをHTMLに埋め込むだけで使えます。

<div id="disqus_thread"></div>
<script>
  var disqus_config = function () {
    this.page.url = window.location.href;  
    this.page.identifier = 'your-page-identifier'; 
  };
  (function() {
    var d = document, s = d.createElement('script');
    s.src = 'https://your-disqus-shortname.disqus.com/embed.js';
    s.setAttribute('data-timestamp', +new Date());
    (d.head || d.body).appendChild(s);
  })();
</script>

公式はこちら → Disqus

例2：問い合わせフォーム（Formspree）

Formspreeはサーバー不要でメールフォームが作れるサービスです。以下のHTMLを使います。

<form action="https://formspree.io/f/your-form-id" method="POST">
  <label>名前: <input type="text" name="name" required></label><br>
  <label>メール: <input type="email" name="_replyto" required></label><br>
  <label>メッセージ: <textarea name="message" required></textarea></label><br>
  <button type="submit">送信</button>
</form>

公式はこちら → Formspree

図解イメージ 📈

以下の流れで静的サイトが生成・配信されます。

MarkdownやHTMLファイル  →  静的サイトジェネレーター（Jekyll, Next.jsなど）
       ↓
生成されたHTML, CSS, JSファイル
       ↓
GitHub/Cloudflare Pagesにデプロイ
       ↓
CDNを通じて高速配信

まとめ 🌟

  • GitHub Pagesは初心者向けで、JekyllでMarkdownがすぐ使える。
  • Cloudflare Pagesはモダンなフレームワーク対応で柔軟かつ高速。
  • Markdown内でHTMLやJavaScriptを埋め込めるので、自由度は思ったより高い。
  • 静的サイトはセキュリティと速度重視、動的機能は外部サービスやサーバーレスで補うのがベター。
